# História 01 — Criar README do projeto

**Como** pessoa desenvolvedora iniciante,  
**quero** documentar o projeto,  
**para** facilitar a execução e entendimento da aplicação.

## Conteúdo sugerido

- Objetivo do projeto
- Tecnologias utilizadas
- Como executar a aplicação
- Como acessar o H2 Console
- Lista de endpoints
- Exemplos de requests
- Estrutura de pacotes

## Critérios de aceite

- Deve existir um arquivo `README.md`.
- O README deve explicar como rodar o projeto localmente.
- O README deve listar os principais endpoints.
- O README deve indicar como acessar o H2 Console.
